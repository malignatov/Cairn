# ideas

A retained spark — a thought worth keeping but **not yet** a project,
decision, or interest. An idea is pre-commitment: it has no goal, no kill
criterion, no commitment. Those are what it might *acquire* if it matures
into a project.

The defining feature of this layer is not capture (everything in Cairn
captures) but **maturation**. Every idea bucket ever built rots — sparks
accumulate, never get revisited, and within a year you have a graveyard.
What prevents that here is a single rule: **ideas must move or die.** An
idea either warms (you keep returning to it; it eventually graduates into a
project) or it cools (you logged it and forgot it). A cooling idea surfaces
itself and asks to be promoted, deliberately parked, or released. This
anti-rot mechanism is why ideas belong in Cairn rather than a notes app:
an idea here isn't just stored, it's watched for whether it's alive.

## Lifecycle

```
spark      → just captured, raw, unevaluated
exploring  → actively being turned over
someday    → deliberately parked (a conscious choice, not neglect)
promoted   → graduated into a project (terminal)
released   → let go (terminal, with optional reason)
```

The crucial distinction is **someday vs. drift**. An idea parked as
`someday` was a deliberate decision — "keep this, but I'm not working on
it." The scanner leaves these alone, forever. An idea sitting in `spark` or
`exploring` and untouched for months is drifted-into-neglect — that's what
the scanner surfaces. Deliberate parking is fine; silent rot is what gets
caught.

## Fields

- `id` — uuid, generated by the server.
- `title` *(required)* — short handle, e.g. "Bio fuse design studio".
- `body` — the fuller thought, in the user's own words. Captured
  faithfully, never standardized into corporate-speak.
- `status` — `spark` (default) | `exploring` | `someday` | `promoted` |
  `released`. Moves along the live ladder (spark ↔ exploring ↔ someday) go
  through `state_update`; the two terminal states go through their own
  tools (`promote_idea` / `release_idea`) because they carry side effects.
- `captured_at` — when the spark was caught.
- `last_touched_at` — bumped whenever the idea is edited, revisited, or
  referenced (`touch_idea`, any `update`, promote, release). This is the
  single signal that drives heat.
- `promoted_to_project_id` — FK to the project the idea became, set on
  promotion; null otherwise.
- `released_reason` — optional, set on release.
- `created_at` — set by the server.
- SLU tags via the shared `entity_slu_links` table (`idea` is a valid link
  entity_type). Most ideas don't map cleanly to a life area — tag only when
  the connection is obvious.

## Heat (derived, never stored)

Computed at read time from `last_touched_at` + `status`:

- `warm` — touched within the last 30 days.
- `neutral` — touched 30 days to 6 months ago.
- `cooling` — untouched for 6+ months **and** status is `spark` or
  `exploring`. A `someday`, `promoted`, or `released` idea never reads
  `cooling`.

There is no reference-counting in v8 — `last_touched_at` is sufficient
signal. Mention-based warming can come later if it proves useful.

## Operations

- Create / read / status-move ride the generic surface (`state_write`,
  `state_query`, `state_update`). Any update touches the idea.
- `state_list_ideas(status?, since?)` — heat-annotated, newest first;
  `status` accepts a single value or the literal `"active"`
  (spark + exploring + someday).
- `state_list_cooling_ideas(dormant_months=6)` — the scanner's feed; only
  spark/exploring past the threshold, never `someday`.
- `state_touch_idea(id)` — keep heat honest on a revisit.
- `state_promote_idea(id)` — create a project from the idea (title → name,
  body → description), set `promoted`, link, touch. Returns the project.
- `state_release_idea(id, reason?)` — set `released`, touch.

## Notes

- No evaluation at capture time — capturing an idea never triggers
  analysis or viability checks. Evaluation is deliberate and separate
  (`review_ideas`, `opportunity_research`).
- No auto-promotion — ideas become projects only by explicit user choice.
- A clean ideas list with a dozen living sparks is worth more than a
  graveyard of three hundred. Releasing is healthy, not failure.
